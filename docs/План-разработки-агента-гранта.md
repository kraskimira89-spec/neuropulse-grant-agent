# План разработки агента-ассистента при реализации гранта

Краткий порядок действий и что уже сделано в проекте.

---

## Порядок действий

| Шаг | Действие | Статус в проекте |
|-----|----------|-------------------|
| 1 | Заполнить Vector Store документами гранта и подключить индекс к агенту | ✅ Вы сделали в Yandex Studio |
| 2 | Доработать инструкцию агента под этапы, сроки и термины | ✅ Вы уточнили в консоли |
| 3 | Веб-чат для команды (без терминала и консоли Yandex) | ✅ Реализован (Streamlit) |
| 4 | Инструменты под типовые запросы по гранту | ✅ Кнопки быстрых действий + конфиг |
| 5 | Напоминания о сроках и помощь в отчётах | ✅ Скрипт напоминаний + кнопки в чате |

## 1. Веб-чат для участников гранта

**Запуск (из корня проекта, с активированным `.venv`):**

```bash
pip install -r requirements.txt
streamlit run src/web_chat.py
```

Откроется браузер с чатом. Все участники гранта могут пользоваться ассистентом без терминала и без доступа к консоли Yandex.

**Возможности:**

- **Новая тема** — начать новый диалог (новая сессия на стороне агента).
- **Продолжить прошлый разговор** — подхватить последнюю сохранённую сессию (общий контекст для команды).
- История диалога в текущей вкладке; контекст хранится в Yandex (Conversations API).
- **Быстрые действия** — кнопки в боковой панели отправляют типовые запросы (срок отчёта, список документов, шаблон отчёта и т.д.).

## 2. Инструменты (быстрые действия) под грант

В **config/config.json** в секции **`grant_tools`** заданы кнопки и промпты:

- Срок следующего отчёта
- Список обязательных документов
- Шаблон отчёта по этапу
- Черновик раздела «Выполненные работы»
- Какие метрики указать по положению

Кнопки отображаются в боковой панели веб-чата; по нажатию запрос уходит агенту (с учётом базы знаний и инструкции). Список можно править в `config.json`: добавлять или менять `id`, `title`, `prompt`.

При необходимости часть сценариев можно вынести в **function calling** (описание функций в инструкции агента и вызов их из кода по ответу модели) — тогда агент сам будет предлагать «узнать срок отчёта» и т.п.

## 3. Напоминания и контроль сроков

**Скрипт:** `scripts/grant_reminders.py`

Читает календарь гранта из **data/grant_calendar.json** (даты, названия, описания) и выводит в консоль ближайшие сроки (по умолчанию на 30 дней вперёд).

**Подготовка:**

1. Скопировать пример: `data/grant_calendar.example.json` → `data/grant_calendar.json`.
2. Заполнить даты отчётов и этапов по положению о гранте.

**Запуск вручную:**

```bash
python scripts/grant_reminders.py
```

**По расписанию:** добавить задачу в планировщик (cron на Linux/macOS или Task Scheduler в Windows), например раз в день. Вывод можно перенаправить в файл или отправить по почте своим скриптом.

Агент в чате может подсказывать формулировки для отчётов и чек-листы; сами напоминания о сроках удобно держать в календаре и в этом скрипте.

## 4. Отчётность и аудит

- **Помощь в текстах отчётов** — через веб-чат и быстрые действия («черновик раздела “Выполненные работы”», «какие метрики указать» и т.д.). Агент опирается на базу знаний и инструкцию.
- **Логирование для аудита** — в веб-чате каждый запрос/ответ логируется в **data/chat_audit.jsonl** без текста сообщений: время (UTC), хеш идентификатора сессии, длина запроса и ответа. Это подходит для внутреннего учёта и отчётности перед организаторами, если условия гранта это допускают.

Файл `data/chat_audit.jsonl` добавлен в `.gitignore`; при необходимости его можно хранить отдельно для аудита.

## 5. Что можно развивать дальше

- **Function calling** — описать в инструкции агента функции («узнать срок отчёта», «выдать список документов») и в коде обрабатывать вызовы этих функций (например, подставлять данные из календаря или конфига).
- **Отдельные страницы/скрипты** под «шаблон отчёта по этапу X» — генерация файла или формы по шаблону, ссылка на которую может быть в чате или в быстрых действиях.
- **Интеграция напоминаний** с почтой или мессенджером (отправка списка сроков командой по расписанию).
